## RESET Pin Configuration

By default, the Reset pin (P0.18) on nRF52840 microcontrollers is configured
as a GPIO. This means that the Reset button does not do anything without
additional configuration. For the nRF52840 DK, the reset pin is automatically
configured by the build system when using the built-in or an external
J-Link programmer.

## Flashing

The various options to flash the nRF52840 DK are explained in
@ref boards_common_nrf52.

## Accessing STDIO via UART

The STDIO is directly accessible via the USB port through the onboard
J-Link programmer. On a Linux host, it is generally mapped to `/dev/ttyACM0`.

Use the `term` target to connect to the board serial port<br/>
```
    make BOARD=nrf52840dk -C examples/hello-world term
```

## Using the Onboard Flash Memory

The nRF52840 DK is equipped with a 64MBit Macronix MX25R6435F NOR Flash chip,
which is supported by the @ref drivers_mtd_spi_nor subsystem.
To get started with the flash, the filesystem example can be used:
```
    make BOARD=nrf52840dk -C examples/filesystem flash term
```
